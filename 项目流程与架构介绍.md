# 工业级智能 Agent 平台 — 项目流程与架构介绍

---

## 一、项目定位

多租户工业级智能 Agent 平台，融合**传统工具调用**与**LLM 动态自研技能**机制。
内置完整的多轮对话、分层记忆、用户画像、动态 Skill 生成、安全沙箱执行、全链路日志监控能力。

### 核心差异化能力

| 特性 | 说明 |
|------|------|
| **双工具体系** | 全局预置公共工具 + 用户私有动态自研 Skill |
| **自主技能迭代** | LLM 可现场编写代码完成复杂任务，支持临时脚本执行 + 通用技能持久化复用 |
| **三级分层记忆** | 短时会话记忆 → 中期摘要记忆 → 长期向量记忆 |
| **精细化用户体系** | 多租户隔离、独立用户画像、私有 Skill 数据隔离 |
| **工业级安全** | 独立远端代码沙箱、高危代码拦截、权限分级管控 |
| **全链路可观测** | TraceID 串联全流程日志、独立异常故障存储、报错溯源复盘 |

## 三、核心业务流程

### 3.1 一次完整的对话流程

```
用户提问
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  1. API 网关                                         │
│     ├─ AuthMiddleware 认证（Bearer Token）            │
│     ├─ TraceIDMiddleware 生成唯一追踪 ID              │
│     └─ 路由分发到 ChatController                     │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│  2. Agent 初始化                                     │
│     ├─ 加载用户画像和偏好设置                         │
│     ├─ 初始化三级记忆编排器                           │
│     │   ├─ ShortTermMemory（Redis/内存）              │
│     │   ├─ SummaryMemory（SQLite）                   │
│     │   └─ LongTermMemory（Qdrant/内存）             │
│     ├─ 检索长期记忆和私有 Skill                       │
│     └─ 构建系统提示词                                 │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│  3. ReAct 循环（最多 5 轮）                          │
│                                                      │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐       │
│  │ 思考     │───►│ 行动     │───►│ 观察     │──┐    │
│  │ LLM 推理 │    │ 工具调用 │    │ 结果分析 │  │    │
│  └──────────┘    └──────────┘    └──────────┘  │    │
│       ▲                                         │    │
│       └─────────────────────────────────────────┘    │
│                                                      │
│  每轮可能执行的操作：                                 │
│  ├─ query_weather      → 预置工具：查天气            │
│  ├─ text_process       → 预置工具：文本处理          │
│  ├─ date_calc          → 预置工具：日期计算          │
│  ├─ execute_python_code → 代码沙箱：生成执行代码     │
│  └─ 用户私有 Skill      → 持久化动态技能            │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│  4. 后期处理                                         │
│     ├─ 将最终回复存入短时记忆                         │
│     ├─ 检查是否需要生成会话摘要（超阈值时触发）       │
│     ├─ 提取长期记忆（关键信息 → 向量库）             │
│     ├─ 判断临时代码是否可持久化为 Skill              │
│     ├─ 更新用户画像                                  │
│     └─ 返回结果给前端                                │
└─────────────────────────────────────────────────────┘
```

```

    │
    ▼

│ └─ 拦截 → 返回错误信息              │
└──────────┬──────────────────────────┘
           │ 通过
           ▼
┌─────────────────────────────────────┐
│ 沙箱执行（sandbox/client.py）       │
│ ├─ mock 模式：临时子进程             │
│ ├─ docker 模式：独立容器（TODO）     │
│ └─ e2b 模式：云端沙箱（TODO）        │
└──────────┬──────────────────────────┘
           │ 执行完毕
           ▼
┌─────────────────────────────────────┐

└─────────────────────────────────────┘
```

## 四、模块详解

### 4.1 多租户用户体系

**数据隔离**：所有用户数据通过 `user_id` 严格分区

| 角色 | 权限 |
|------|------|
| `normal` | 仅调用全局预置工具 |
| `advanced` | 开放代码沙箱 + 动态 Skill |
| `admin` | 全量数据 + 日志审计 + 用户管理 |

### 4.2 三级分层记忆

| 层级 | 存储 | 生命周期 | 用途 |
|------|------|----------|------|
| 短时记忆 | Redis / 内存 | 会话 | 当前对话窗口，滚动截断 |
| 摘要记忆 | SQLite | 会话 | Token 超阈值时 LLM 压缩生成 |
| 长期记忆 | Qdrant / 内存向量 | 跨会话 | 语义检索历史关键信息 |

### 4.3 预置工具

| 工具名 | 功能 | 文件 |
|--------|------|------|
| `query_weather` | 查天气（mock） | `tools/preset/weather.py` |
| `text_process` | 文本处理 | `tools/preset/text_processing.py` |
| `date_calc` | 日期计算 | `tools/preset/date_calc.py` |

### 4.4 代码沙箱安全

**3 层防护**：
1. **静态扫描**（`code_scanner.py`）：18 条危险规则，匹配 `os.remove`、`subprocess`、`eval`、`ctypes` 等
2. **隔离执行**（`sandbox/client.py`）：临时子进程 / Docker 容器，超时自动终止
3. **全量审计**：所有代码、执行结果、报错均关联 TraceID 入库

### 4.5 第三方依赖自动降级

| 服务 | 首选 | 降级方案 |
|------|------|----------|
| Redis | `redis.Redis` | `_InMemoryRedis`（字典） |
| Qdrant | `QdrantClient` | `_InMemoryVectorStore`（暴力余弦检索） |

## 七、启动与使用

```bash
# 激活环境
conda activate crafto

# 配置 API Key（编辑 .env 或 export）
export DEEPSEEK_API_KEY=sk-your-key-here

# 启动服务
cd /Users/ye/code/crafto
uvicorn app.main:app --host 0.0.0.0 --port 8100 --reload

# 打开浏览器
open http://localhost:8100
```

### 测试接口

```bash
# 注册
curl -X POST http://localhost:8100/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456","role":"advanced"}'

# 对话
curl -X POST http://localhost:8100/api/v1/chat/send \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {user_id}" \
  -d '{"message":"今天广州的天气怎么样？"}'
```
